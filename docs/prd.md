# **PRD: 좌석 예약 시스템 (MVP)**

## 1. 제품 개요

사용자가 특정 공연의 좌석을 선택하고 예매할 수 있는 경량 좌석 예약 시스템.
**핵심 목표**는 (1) 다수 동시 접속 환경에서 특정 좌석에 대한 **중복 예약을 방지**하는 안정적인 선점 처리 로직 구현, (2) 사용자가 좌석 현황 파악부터 예약 완료까지 최소한의 단계로 진행할 수 있는 **직관적인 UI/UX 설계**이다. MVP 범위에서는 별도의 **인증/로그인 기능을 제외**하여, 비회원 기반의 빠른 예약 플로우에 집중한다.

## 2. Stakeholders

*   **예매자(User)**: 콘서트 탐색, 좌석 선택/예약, 자신의 예약 내역 조회/취소
*   **운영자(Operator)**: (별도 관리 페이지 없이) DB를 통해 직접 공연 및 좌석 정보 등록/관리

## 3. 포함 페이지

1.  **홈**
    *   예약 가능한 콘서트 목록을 카드로 노출

2.  **콘서트 상세**
    *   히어로 섹션 이미지, 공연 일시 및 장소
    *   등급별 가격 및 잔여 좌석 수 안내
    *   전체 예약 현황 (예: 잔여 150석 / 전체 500석)
    *   '예약하기' 버튼 (좌석 선택 페이지로 이동)

3.  **좌석 선택**
    *   콘서트 제목 및 전체 좌석 현황
    *   등급별 가격/좌석 정보 안내 카드 *(예: 스페셜(1-3열) | 250,000원 | 48/48석)*
    *   구역 및 등급별 색상으로 구분된 좌석 배치도
    *   선택한 좌석 정보를 보여주는 실시간 현황 카드
    *   선택한 좌석 수에 따라 동적으로 변하는 '예약하기 (N석 선택)' 버튼

4.  **좌석 선점 및 예약 정책**
    *   사용자가 '좌석 선택' 페이지에서 특정 좌석을 선택하면, 해당 좌석은 **5분간 '임시 선점' 상태**로 변경되어 다른 사용자가 선택할 수 없다.
    *   5분 내에 '예약 정보 입력'을 완료하지 않으면 선점은 **자동으로 해제**되어 다른 사용자가 선택할 수 있게 된다.

5.  **예약 정보 입력**
    *   선택한 좌석 정보(구역, 번호, 가격) 재확인
    *   예약자 정보 입력 폼 (이름, 휴대폰 번호, 비밀번호)
    *   '예약 완료' 버튼

6.  **예약 완료**
    *   예약 성공 메시지 및 고유 예약 번호
    *   예약 정보 상세 (공연 정보, 좌석, 총액)
    *   '홈으로 가기', '예약 조회하기' 버튼

7.  **예약 조회 (입력폼)**
    *   예약 내역 조회를 위한 정보 입력 폼 (휴대폰 번호, 비밀번호)
    *   '예약 조회' 버튼

8.  **예약 내역 상세 (조회 결과)**
    *   예약 정보 전체 내역 표시
    *   '예약 취소' 버튼

## 4. 사용자 여정 (User Journey)

### 4.1 예매자 여정 (좌석 예매)

1.  홈에서 콘서트 탐색 → 콘서트 상세 페이지로 이동
2.  잔여 좌석 확인 후 → '예약하기' 버튼 클릭
3.  좌석 배치도에서 좌석 선택 → '예약하기 (N석 선택)' 버튼 클릭
4.  예약자 정보(이름, 연락처, 비밀번호) 입력 → '예약 완료' 버튼 클릭
5.  예약 완료 페이지에서 예약 번호 및 상세 내역 확인

### 4.2 예매자 여정 (조회 및 취소)

1.  예약 조회 페이지로 이동 → 휴대폰 번호/비밀번호 입력 후 조회
2.  예약 내역 상세 페이지에서 정보 확인
3.  필요시 '예약 취소' 버튼 클릭

## 5. IA (Information Architecture)

```
/ (Home)
 └─ # 메인 랜딩 페이지, 콘서트 목록(카드 리스트) 노출

/concerts
 └─ /[concertId]
    ├─ # (GET) 콘서트 상세 정보 페이지
    │  # 히어로 이미지, 공연 정보, 잔여석 현황 등을 표시
    └─ /seats
       └─ # (GET) 좌석 선택 페이지
          # 좌석 배치도를 표시하고 사용자의 좌석 선택을 처리

/book
 └─ # (GET, POST) 예약 정보 입력 및 제출 페이지
    # '/concerts/[concertId]/seats' 에서 선택한 좌석 정보를 받아
    # 예약자 정보(이름, 연락처, 비밀번호) 입력 폼을 제공하고, 제출 시 예약을 생성

/reservations
 ├─ /lookup
 │  └─ # (GET, POST) 예약 조회 페이지
 │     # 예약자 정보(연락처, 비밀번호)를 입력받아 일치하는 예약을 검색
 ├─ /complete
 │  └─ # (GET) 예약 완료 페이지
 │     # 예약 성공 후 리다이렉트 되는 페이지. 예약 번호와 상세 정보를 표시
 └─ /[reservationId]
    └─ # (GET, DELETE) 예약 내역 상세 페이지
       # 조회된 특정 예약의 전체 정보를 표시하고, 예약 취소(DELETE) 기능을 제공
```
